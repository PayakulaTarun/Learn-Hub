HTML Semantic Elements

Theory & Concepts

What is Semantics?

Semantics is the study of meaning. A semantic element clearly describes its meaning to both the browser and the developer.

Non-Semantic vs Semantic

  • Non-Semantic Elements: <div> and <span>. Tells nothing about its content. Just a container.
  • Semantic Elements: <form>, <table>, <article>. Clearly defines its content.

HTML5 Semantic Elements

HTML5 introduced new tags to define the structure of a page more clearly:

  • <header>: Introductory content or navigation links at the top.
  • <nav>: A set of navigation links.
  • <main>: The dominant content of the document (should be unique).
  • <section>: A thematic grouping of content, typically with a heading.
  • <article>: Independent, self-contained content (e.g., a blog post, news story).
  • <aside>: Content aside from the page content (sidebar).
  • <footer>: Footer for a document or section (copyright, contact info).

Why Use Semantic Tags?

  1. Accessibility: Screen readers can navigate directly to the "Main" content or "Nav".
  2. SEO: Search engines understand which parts are important content vs boilerplate (header/footer).
  3. Readability: Code is easier for humans to read.
