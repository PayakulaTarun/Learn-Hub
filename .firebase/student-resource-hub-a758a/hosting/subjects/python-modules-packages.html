Python Modules and Packages - Complete Guide

python • Knowledge Pillar

Theory & Concepts

What are Modules?

Module is a Python file (.py) containing code (functions, classes, variables) that you can import and use in other files.

Real-Life Analogy

Think of tool boxes:

  • Module: Toolbox with specific tools (math tools, file tools, etc.)
  • Import: Taking toolbox from storage to use
  • Package: Cabinet with multiple related toolboxes

You don't rebuild tools - you import and use them!

Why Use Modules?

  1. Code Reusability: Write once, use many times
  2. Organization: Break large code into manageable pieces
  3. Namespace: Avoid naming conflicts
  4. Maintainability: Easier to update and debug
  5. Collaboration: Share code with others

Types of Modules

  1. Built-in: Comes with Python (math, os, sys, datetime)
  2. Third-party: Install via pip (requests, numpy, pandas)
  3. Custom: Your own .py files

Importing Modules

Method 1: import module

import math

result = math.sqrt(16)
print(result)  # 4.0
print(math.pi)  # 3.141592653589793

Use: When you need many items from module.

Method 2: from module import item

from math import sqrt, pi

result = sqrt(16)  # No math. prefix needed
print(result)  # 4.0
print(pi)  # 3.141592653589793

Use: When you need specific items.

Method 3: from module import *

from math import *

result = sqrt(16)  # All math functions available
print(pi)

Warning: Can cause naming conflicts. Use sparingly!

Method 4: import module as alias

import numpy as np  # Common convention
import pandas as pd

array = np.array([1, 2, 3])
df = pd.DataFrame({'A': [1, 2, 3]})

Use: Shorter names for frequently used modules.

Method 5: from module import item as alias

from math import sqrt as square_root

result = square_root(16)
print(result)  # 4.0

Import Summary Table

| Syntax | Access | Example | |--------|--------|----------| | import math | math.sqrt() | Need full path | | from math import sqrt | sqrt() | Direct access | | from math import * | sqrt() | All items (avoid) | | import numpy as np | np.array() | Shorter alias | | from math import sqrt as sq | sq() | Rename item |

Built-in Modules

Python comes with 200+ built-in modules!

Commonly Used Built-in Modules

math: Mathematical functions

import math

print(math.sqrt(16))      # 4.0
print(math.factorial(5))  # 120
print(math.ceil(4.3))     # 5
print(math.floor(4.9))    # 4

random: Random number generation

import random

print(random.randint(1, 10))      # Random int 1-10
print(random.choice(['a', 'b', 'c']))  # Random choice
print(random.random())            # Random float 0-1

datetime: Date and time

from datetime import datetime, timedelta

now = datetime.now()
print(now.strftime('%Y-%m-%d %H:%M:%S'))

tomorrow = now + timedelta(days=1)
print(tomorrow)

os: Operating system functions

import os

print(os.getcwd())           # Current directory
print(os.listdir('.'))       # List files
os.mkdir('new_folder')       # Create directory
print(os.path.exists('file.txt'))  # Check if exists

sys: System-specific parameters

import sys

print(sys.version)           # Python version
print(sys.platform)          # Operating system
print(sys.path)              # Module search path

json: JSON encoding/decoding

import json

data = {'name': 'Alice', 'age': 25}
json_string = json.dumps(data)
print(json_string)  # {"name": "Alice", "age": 25}

data_back = json.loads(json_string)
print(data_back['name'])  # Alice

Creating Custom Modules

Step 1: Create Module File

File: math_utils.py

"""Math utility functions"""

def add(a, b):
    """Add two numbers"""
    return a + b

def multiply(a, b):
    """Multiply two numbers"""
    return a * b

def factorial(n):
    """Calculate factorial"""
    if n == 0 or n == 1:
        return 1
    return n * factorial(n - 1)

# Module-level variable
PI = 3.14159

Step 2: Import and Use

File: main.py

import math_utils

result = math_utils.add(5, 3)
print(result)  # 8

print(math_utils.PI)  # 3.14159
print(math_utils.factorial(5))  # 120

if name == "main"

Code that should only run when file is executed directly:

File: math_utils.py

def add(a, b):
    return a + b

# Only runs when file executed directly
if __name__ == "__main__":
    # Test code
    print("Testing add function:")
    print(add(2, 3))  # 5

Execution:

  • python math_utils.py → Test code runs
  • import math_utils → Test code doesn't run

Python Packages

Package is a directory containing multiple modules and a special __init__.py file.

Package Structure

my_package/
    __init__.py
    module1.py
    module2.py
    subpackage/
        __init__.py
        module3.py

Creating a Package

Directory structure:

math_package/
    __init__.py
    basic.py
    advanced.py

File: math_package/basic.py

def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

File: math_package/advanced.py

import math

def power(base, exp):
    return base ** exp

def sqrt(n):
    return math.sqrt(n)

File: math_package/init.py

# This makes it a package
# Can be empty or contain initialization code

from .basic import add, subtract
from .advanced import power, sqrt

__version__ = '1.0.0'

Using the Package

# Import entire package
import math_package

print(math_package.add(5, 3))  # 8
print(math_package.sqrt(16))   # 4.0

# Import specific module
from math_package import basic

print(basic.add(10, 5))  # 15

# Import specific function
from math_package.advanced import power

print(power(2, 8))  # 256

Module Search Path

When you import, Python searches in order:

  1. Current directory
  2. PYTHONPATH (environment variable)
  3. Standard library
  4. Site-packages (third-party modules)

Viewing Search Path

import sys

for path in sys.path:
    print(path)

Adding to Path

import sys

sys.path.append('/path/to/my/modules')
import my_module

pip - Python Package Manager

pip installs third-party packages from PyPI (Python Package Index).

Basic pip Commands

Install package:

pip install package_name
pip install requests
pip install numpy pandas matplotlib

Install specific version:

pip install package_name==version
pip install requests==2.28.0

Upgrade package:

pip install --upgrade package_name
pip install --upgrade requests

Uninstall package:

pip uninstall package_name
pip uninstall requests

List installed packages:

pip list

Show package info:

pip show package_name
pip show requests

Check outdated packages:

pip list --outdated

requirements.txt

File listing all dependencies:

requirements.txt:

requests==2.28.0
numpy>=1.20.0
pandas>=1.3.0
matplotlib

Install all requirements:

pip install -r requirements.txt

Generate requirements.txt:

pip freeze > requirements.txt

Virtual Environments

Virtual environment isolates project dependencies.

Why Virtual Environments?

  • Different projects need different package versions
  • Avoid conflicts between projects
  • Clean, reproducible environments

Creating Virtual Environment

Using venv (built-in):

# Create
python -m venv myenv

# Activate (Windows)
myenv\Scripts\activate

# Activate (Mac/Linux)
source myenv/bin/activate

# Deactivate
deactivate

Workflow

# 1. Create virtual environment
python -m venv venv

# 2. Activate
venv\Scripts\activate  # Windows

# 3. Install packages
pip install requests pandas

# 4. Save requirements
pip freeze > requirements.txt

# 5. Work on project...

# 6. Deactivate when done
deactivate

Relative vs Absolute Imports

Project Structure

my_package/
    __init__.py
    module_a.py
    module_b.py
    subpackage/
        __init__.py
        module_c.py

Absolute Imports (Recommended)

# In module_c.py
from my_package import module_a
from my_package.module_b import some_function

Relative Imports

# In module_c.py
from .. import module_a       # Parent directory
from ..module_b import func   # Parent directory
from . import module_d        # Same directory

Recommendation: Use absolute imports for clarity.

Creating Installable Package

setup.py

from setuptools import setup, find_packages

setup(
    name='my_package',
    version='1.0.0',
    author='Your Name',
    author_email='you@example.com',
    description='A sample package',
    packages=find_packages(),
    install_requires=[
        'requests>=2.25.0',
        'numpy>=1.20.0',
    ],
    python_requires='>=3.6',
)

Install in Development Mode

pip install -e .

Build and Distribute

# Build
python setup.py sdist bdist_wheel

# Upload to PyPI
pip install twine
twine upload dist/*

Common Module Patterns

Singleton Module

config.py:

class Config:
    DEBUG = True
    DATABASE_URL = 'sqlite:///db.sqlite3'

config = Config()

Usage:

from config import config

if config.DEBUG:
    print("Debug mode")

Factory Module

factory.py:

def create_user(user_type):
    if user_type == 'admin':
        return AdminUser()
    elif user_type == 'regular':
        return RegularUser()

Constants Module

constants.py:

MAX_CONNECTIONS = 100
DEFAULT_TIMEOUT = 30
API_BASE_URL = 'https://api.example.com'
