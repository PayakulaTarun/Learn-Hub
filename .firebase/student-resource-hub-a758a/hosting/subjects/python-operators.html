Theory & Concepts

What are Operators?

Operators are special symbols that perform operations on operands (values or variables).

Example: In 5 + 3, + is the operator, 5 and 3 are operands.

Types of Operators in Python

  1. Arithmetic Operators
  2. Relational (Comparison) Operators
  3. Logical Operators
  4. Assignment Operators
  5. Membership Operators
  6. Identity Operators
  7. Bitwise Operators (Advanced)

1. Arithmetic Operators

Perform mathematical calculations.

Operator Table:

| Operator | Name | Example | Result | |----------|------|---------|--------| | + | Addition | 5 + 3 | 8 | | - | Subtraction | 5 - 3 | 2 | | * | Multiplication | 5 * 3 | 15 | | / | Division (float) | 7 / 2 | 3.5 | | // | Floor Division | 7 // 2 | 3 | | % | Modulus (Remainder) | 7 % 3 | 1 | | ** | Exponentiation | 2 ** 3 | 8 |

Division Types

Regular Division /: Always returns float

10 / 3   # 3.333...
10 / 2   # 5.0 (still float!)

Floor Division //: Returns integer (rounds down)

10 // 3  # 3
-10 // 3 # -4 (rounds down toward negative infinity)

Modulus %: Returns remainder

10 % 3   # 1 (10 = 3*3 + 1)
10 % 2   # 0 (even number check)

Exponentiation **: Power operation

2 ** 3   # 8 (2³)
5 ** 2   # 25 (5²)
4 ** 0.5 # 2.0 (square root)

Unary Operators

+x   # Positive (rarely used)
-x   # Negative

2. Relational (Comparison) Operators

Compare values and return boolean (True/False).

Operator Table:

| Operator | Name | Example | Result | |----------|------|---------|--------| | == | Equal to | 5 == 5 | True | | != | Not equal to | 5 != 3 | True | | > | Greater than | 5 > 3 | True | | < | Less than | 5 < 3 | False | | >= | Greater or equal | 5 >= 5 | True | | <= | Less or equal | 3 <= 5 | True |

Important Notes

== vs =:

  • = assigns value (x = 5)
  • == compares values (x == 5)

String Comparison:

"apple" < "banana"  # True (alphabetical order)
"ABC" < "abc"       # True (uppercase < lowercase in ASCII)

Chaining Comparisons (Python feature!):

x = 5
1 < x < 10  # True (same as: 1 < x and x < 10)

3. Logical Operators

Combine multiple conditions.

Operator Table:

| Operator | Description | Example | Result | |----------|-------------|---------|--------| | and | True if BOTH are true | True and False | False | | or | True if AT LEAST ONE is true | True or False | True | | not | Inverts boolean | not True | False |

Truth Tables

AND (and) - Both must be True:

True  and True   → True
True  and False  → False
False and True   → False
False and False  → False

OR (or) - At least one must be True:

True  or True   → True
True  or False  → True
False or True   → True
False or False  → False

NOT (not) - Inverts the value:

not True   → False
not False  → True

Practical Examples

age = 25
has_license = True

# Can drive if: age >= 18 AND has license
can_drive = (age >= 18) and has_license  # True

# Discount if: student OR senior
is_student = True
is_senior = False
gets_discount = is_student or is_senior  # True

# Not logged in
is_logged_in = False
show_login_page = not is_logged_in  # True

Short-Circuit Evaluation

Python stops evaluating as soon as result is determined.

# AND: If first is False, second isn't checked
False and expensive_function()  # expensive_function() never runs

# OR: If first is True, second isn't checked
True or expensive_function()    # expensive_function() never runs

4. Assignment Operators

Assign values to variables.

Operator Table:

| Operator | Example | Equivalent To | |----------|---------|---------------| | = | x = 5 | x = 5 | | += | x += 3 | x = x + 3 | | -= | x -= 3 | x = x - 3 | | *= | x *= 3 | x = x * 3 | | /= | x /= 3 | x = x / 3 | | //= | x //= 3 | x = x // 3 | | %= | x %= 3 | x = x % 3 | | **= | x **= 3 | x = x ** 3 |

Augmented Assignment Examples

count = 10
count += 5   # count = 15
count -= 3   # count = 12
count *= 2   # count = 24
count //= 4  # count = 6

Multiple Assignment

# Assign same value to multiple variables
a = b = c = 0

# Unpack multiple values
x, y, z = 1, 2, 3

# Swap values (Python magic!)
a, b = b, a

5. Membership Operators

Check if value exists in sequence (string, list, tuple, set, dict).

Operator Table:

| Operator | Description | Example | Result | |----------|-------------|---------|--------| | in | Returns True if value found | 5 in [1,2,5] | True | | not in | Returns True if value NOT found | 5 not in [1,2,3] | True |

Examples

Lists and Tuples:

numbers = [1, 2, 3, 4, 5]
3 in numbers      # True
10 in numbers     # False
10 not in numbers # True

Strings:

text = "Hello, World!"
"Hello" in text     # True
"hello" in text     # False (case-sensitive!)
"x" not in text     # True

Dictionaries (checks keys, not values):

user = {"name": "Alice", "age": 25}
"name" in user      # True
"Alice" in user     # False (it's a value, not key)
"email" not in user # True

Sets:

fruits = {"apple", "banana", "orange"}
"apple" in fruits      # True
"grape" not in fruits  # True

6. Identity Operators

Check if two variables refer to the same object in memory (not just equal values).

Operator Table:

| Operator | Description | Example | |----------|-------------|---------| | is | True if same object | a is b | | is not | True if different objects | a is not b |

is vs ==

==: Compares values is: Compares identity (memory location)

x = [1, 2, 3]
y = [1, 2, 3]
z = x

print(x == y)  # True (same values)
print(x is y)  # False (different objects)
print(x is z)  # True (same object)

Special Cases

Small integers and strings (cached by Python):

a = 5
b = 5
print(a is b)  # True (Python caches small integers)

x = "hello"
y = "hello"
print(x is y)  # True (Python caches strings)

None (always use is):

value = None
if value is None:      # ✅ Correct
    print("No value")

if value == None:      # ❌ Works but not recommended
    print("No value")

When to Use

  • ✅ Use == for value comparison
  • ✅ Use is only for:
    • Checking against None
    • Checking object identity (rare)

Operator Precedence

Order in which operators are evaluated.

High to Low Priority:

  1. ** Exponentiation
  2. +x, -x, ~x Unary operators
  3. *, /, //, % Multiplication, Division, Modulus
  4. +, - Addition, Subtraction
  5. <, <=, >, >=, !=, == Comparisons
  6. is, is not Identity
  7. in, not in Membership
  8. not Logical NOT
  9. and Logical AND
  10. or Logical OR
  11. =, +=, -=, etc. Assignment

Precedence Examples

# Without parentheses
result = 2 + 3 * 4      # 14 (not 20)
# Multiplication happens first: 3*4=12, then 2+12=14

# With parentheses
result = (2 + 3) * 4    # 20
# Parentheses force addition first: 2+3=5, then 5*4=20

# Complex example
x = 5
result = x > 3 and x < 10  # True
# Comparison first: (x > 3) and (x < 10)
#                    True and True → True

Best Practice

Use parentheses for clarity, even when not required:

# Unclear
if a > b and c < d or e == f:
    pass

# Clear
if ((a > b) and (c < d)) or (e == f):
    pass
