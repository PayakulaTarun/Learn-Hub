HTML Security Basics

Security is often handled on the server, but the frontend HTML plays a big role.

1. XSS (Cross-Site Scripting)

Attackers inject malicious scripts into your web page.

  • Prevention: Never trust user input. Always escape special characters (< to &lt;) before displaying user data.

2. Content Security Policy (CSP)

A meta tag (or header) that tells the browser which sources are allowed to run.

<meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self' https://trusted.com">

This prevents attackers from loading scripts from evil-site.com.

3. Iframe Security

Iframes can be dangerous. Use the sandbox attribute to restrict what the embedded content can do.

<iframe src="untrusted.html" sandbox></iframe>

sandbox bans scripts, forms, and popups by default.

4. HTTPS

Always serve HTML over HTTPS. Unencrypted HTTP allows attackers to modify the HTML in transit (Man-in-the-Middle).
