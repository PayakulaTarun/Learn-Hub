Theory & Concepts

What is Object-Oriented Programming?

OOP is a programming paradigm that organizes code around objects (data + behavior) rather than functions and logic.

Real-World Analogy

Think of car manufacturing:

  • Blueprint (Class): Car design specifications
  • Actual Car (Object): Physical car built from blueprint
  • Properties (Attributes): Color, brand, speed
  • Actions (Methods): start(), accelerate(), brake()

Multiple cars (objects) can be created from same blueprint (class), each with different properties.

Four Pillars of OOP

  1. Encapsulation: Bundle data and methods together
  2. Abstraction: Hide complex implementation details
  3. Inheritance: Reuse code from existing classes
  4. Polymorphism: Same interface, different implementations

Procedural vs OOP

Procedural (Functions):

# Data separated from functions
car1_brand = "Toyota"
car1_speed = 0

def accelerate_car1():
    global car1_speed
    car1_speed += 10

OOP (Classes):

# Data and functions together
class Car:
    def __init__(self, brand):
        self.brand = brand
        self.speed = 0
    
    def accelerate(self):
        self.speed += 10

car1 = Car("Toyota")
car1.accelerate()

Classes and Objects

Class (Blueprint)

Class is a template/blueprint for creating objects.

class Car:
    # Class definition
    pass

Analogy: Cookie cutter is the class, cookies are objects.

Object (Instance)

Object is a specific instance created from a class.

# Create objects from class
car1 = Car()  # Object 1
car2 = Car()  # Object 2

Visual Representation

Class: Car (Blueprint)
  ├── Attributes: brand, color, speed
  └── Methods: start(), stop(), accelerate()

Objects (Instances):
  Car1: brand="Toyota", color="Red", speed=0
  Car2: brand="Honda", color="Blue", speed=0
  Car3: brand="Tesla", color="White", speed=0

Basic Class Example

class Dog:
    # Class attribute (shared by all dogs)
    species = "Canis familiaris"
    
    # Instance method
    def bark(self):
        return "Woof!"

# Create objects
dog1 = Dog()
dog2 = Dog()

print(dog1.species)  # Canis familiaris
print(dog1.bark())   # Woof!

Constructor (init)

Constructor is a special method that initializes object state when created.

Syntax

class ClassName:
    def __init__(self, parameters):
        # Initialize instance attributes
        self.attribute = value

The 'self' Parameter

self refers to the current instance.

class Person:
    def __init__(self, name, age):
        self.name = name  # Instance attribute
        self.age = age    # Instance attribute
    
    def introduce(self):
        return f"Hi, I'm {self.name}, {self.age} years old"

# Create objects
person1 = Person("Alice", 25)
person2 = Person("Bob", 30)

print(person1.introduce())  # Hi, I'm Alice, 25 years old
print(person2.introduce())  # Hi, I'm Bob, 30 years old

Class vs Instance Attributes

class Employee:
    # Class attribute (shared by all)
    company = "TechCorp"
    
    def __init__(self, name, salary):
        # Instance attributes (unique to each)
        self.name = name
        self.salary = salary

emp1 = Employee("Alice", 50000)
emp2 = Employee("Bob", 60000)

print(emp1.company)  # TechCorp (class attribute)
print(emp1.name)     # Alice (instance attribute)
print(emp2.name)     # Bob (different instance)

Instance Methods

Methods that operate on instance data.

class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.balance = balance
    
    def deposit(self, amount):
        self.balance += amount
        return f"Deposited ${amount}. New balance: ${self.balance}"
    
    def withdraw(self, amount):
        if amount > self.balance:
            return "Insufficient funds"
        self.balance -= amount
        return f"Withdrew ${amount}. New balance: ${self.balance}"

account = BankAccount("Alice", 1000)
print(account.deposit(500))   # Deposited $500. New balance: $1500
print(account.withdraw(200))  # Withdrew $200. New balance: $1300

Inheritance

Inheritance allows a class to inherit attributes and methods from another class.

Syntax

class ParentClass:
    # Parent class definition
    pass

class ChildClass(ParentClass):
    # Child class inherits from parent
    pass

Visual Hierarchy

        Animal (Parent)
       /      |      \
     Dog    Cat    Bird (Children)

Basic Inheritance

# Parent class
class Animal:
    def __init__(self, name):
        self.name = name
    
    def speak(self):
        return "Some sound"

# Child class
class Dog(Animal):
    def speak(self):  # Override parent method
        return "Woof!"

class Cat(Animal):
    def speak(self):  # Override parent method
        return "Meow!"

# Create objects
dog = Dog("Buddy")
cat = Cat("Whiskers")

print(dog.name)    # Buddy (inherited from Animal)
print(dog.speak()) # Woof! (overridden in Dog)
print(cat.speak()) # Meow! (overridden in Cat)

super() Function

Call parent class methods:

class Vehicle:
    def __init__(self, brand, year):
        self.brand = brand
        self.year = year
    
    def info(self):
        return f"{self.brand} ({self.year})"

class Car(Vehicle):
    def __init__(self, brand, year, doors):
        super().__init__(brand, year)  # Call parent constructor
        self.doors = doors
    
    def info(self):
        parent_info = super().info()  # Call parent method
        return f"{parent_info}, {self.doors} doors"

car = Car("Toyota", 2024, 4)
print(car.info())  # Toyota (2024), 4 doors

Multiple Inheritance

Inherit from multiple parents:

class Flyable:
    def fly(self):
        return "Flying..."

class Swimmable:
    def swim(self):
        return "Swimming..."

class Duck(Flyable, Swimmable):
    def quack(self):
        return "Quack!"

duck = Duck()
print(duck.fly())   # Flying... (from Flyable)
print(duck.swim())  # Swimming... (from Swimmable)
print(duck.quack()) # Quack! (own method)

Polymorphism

Polymorphism means "many forms" - same interface, different implementations.

Method Overriding

Child class redefines parent method:

class Shape:
    def area(self):
        return 0

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height
    
    def area(self):  # Override
        return self.width * self.height

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    
    def area(self):  # Override
        return 3.14 * self.radius ** 2

# Polymorphic behavior
shapes = [Rectangle(5, 10), Circle(7)]

for shape in shapes:
    print(f"Area: {shape.area()}")  # Different implementations

Duck Typing

If it walks like a duck and quacks like a duck, it's a duck.

class Dog:
    def speak(self):
        return "Woof!"

class Cat:
    def speak(self):
        return "Meow!"

class Robot:
    def speak(self):
        return "Beep!"

def make_speak(thing):
    # Don't care about type, just that it has speak()
    return thing.speak()

print(make_speak(Dog()))    # Woof!
print(make_speak(Cat()))    # Meow!
print(make_speak(Robot()))  # Beep!

Encapsulation

Encapsulation bundles data and methods, hiding internal implementation.

Public, Protected, Private

class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner           # Public
        self._account_number = "1234"  # Protected (convention)
        self.__pin = "0000"           # Private (name mangling)
    
    def get_pin(self):  # Public method to access private
        return "Cannot show PIN"
    
    def __validate_pin(self, pin):  # Private method
        return pin == self.__pin

account = BankAccount("Alice", 1000)
print(account.owner)           # Public - OK
print(account._account_number) # Protected - works but discouraged
# print(account.__pin)         # Private - AttributeError!
print(account.get_pin())       # Access via public method

Naming Conventions:

  • public: Normal attribute/method
  • _protected: Single underscore (convention, not enforced)
  • __private: Double underscore (name mangling)

Getters and Setters

class Person:
    def __init__(self, name, age):
        self._name = name
        self._age = age
    
    # Getter
    @property
    def age(self):
        return self._age
    
    # Setter
    @age.setter
    def age(self, value):
        if value < 0:
            raise ValueError("Age cannot be negative")
        self._age = value

person = Person("Alice", 25)
print(person.age)  # Uses getter: 25

person.age = 26    # Uses setter
print(person.age)  # 26

# person.age = -5  # ValueError!

Why Encapsulation?

  1. Data Protection: Control access to data
  2. Validation: Validate before setting values
  3. Flexibility: Change implementation without breaking code
  4. Abstraction: Hide complexity

Abstraction

Abstraction hides implementation details, shows only essential features.

Abstract Base Classes

from abc import ABC, abstractmethod

class Animal(ABC):  # Abstract class
    @abstractmethod
    def speak(self):
        pass  # Must be implemented by child
    
    @abstractmethod
    def move(self):
        pass  # Must be implemented by child

# Cannot instantiate abstract class
# animal = Animal()  # TypeError!

class Dog(Animal):
    def speak(self):
        return "Woof!"
    
    def move(self):
        return "Running"

# Must implement all abstract methods
dog = Dog()
print(dog.speak())  # Woof!
print(dog.move())   # Running

Interface-like Design

from abc import ABC, abstractmethod

class PaymentProcessor(ABC):
    @abstractmethod
    def process_payment(self, amount):
        pass

class CreditCardProcessor(PaymentProcessor):
    def process_payment(self, amount):
        return f"Processing ${amount} via Credit Card"

class PayPalProcessor(PaymentProcessor):
    def process_payment(self, amount):
        return f"Processing ${amount} via PayPal"

def checkout(processor, amount):
    return processor.process_payment(amount)

print(checkout(CreditCardProcessor(), 100))
print(checkout(PayPalProcessor(), 50))

Special Methods (Magic Methods)

str and repr

class Book:
    def __init__(self, title, author):
        self.title = title
        self.author = author
    
    def __str__(self):  # User-friendly string
        return f"{self.title} by {self.author}"
    
    def __repr__(self):  # Developer-friendly representation
        return f"Book('{self.title}', '{self.author}')"

book = Book("Python Guide", "Alice")
print(str(book))   # Python Guide by Alice
print(repr(book))  # Book('Python Guide', 'Alice')

Operator Overloading

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __add__(self, other):  # + operator
        return Vector(self.x + other.x, self.y + other.y)
    
    def __str__(self):
        return f"Vector({self.x}, {self.y})"

v1 = Vector(2, 3)
v2 = Vector(4, 5)
v3 = v1 + v2  # Calls __add__
print(v3)  # Vector(6, 8)

Other Magic Methods

class MyList:
    def __init__(self, items):
        self.items = items
    
    def __len__(self):  # len()
        return len(self.items)
    
    def __getitem__(self, index):  # []
        return self.items[index]
    
    def __contains__(self, item):  # in
        return item in self.items

my_list = MyList([1, 2, 3, 4, 5])
print(len(my_list))    # 5
print(my_list[2])      # 3
print(3 in my_list)    # True

Class Methods and Static Methods

Class Methods

class Employee:
    company = "TechCorp"
    
    def __init__(self, name):
        self.name = name
    
    @classmethod
    def change_company(cls, new_company):
        cls.company = new_company

# Call on class
Employee.change_company("NewCorp")
print(Employee.company)  # NewCorp

Static Methods

class MathUtils:
    @staticmethod
    def add(a, b):
        return a + b
    
    @staticmethod
    def multiply(a, b):
        return a * b

# Call without instance
print(MathUtils.add(5, 3))       # 8
print(MathUtils.multiply(4, 7))  # 28

Comparison

| Type | Access | Use Case | |------|--------|----------| | Instance method | self | Operate on instance data | | Class method | cls | Operate on class data | | Static method | Neither | Utility functions |

Composition

Composition: "has-a" relationship (alternative to inheritance).

class Engine:
    def start(self):
        return "Engine started"

class Car:
    def __init__(self, brand):
        self.brand = brand
        self.engine = Engine()  # Car HAS an Engine
    
    def start(self):
        return f"{self.brand}: {self.engine.start()}"

car = Car("Toyota")
print(car.start())  # Toyota: Engine started

When to use:

  • Inheritance: "is-a" (Dog IS an Animal)
  • Composition: "has-a" (Car HAS an Engine)
