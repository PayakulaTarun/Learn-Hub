Cross-Browser Compatibility

Not all browsers (Chrome, Firefox, Safari, Edge) work exactly the same way. HTML/CSS code can look different on different browsers.

1. Browser Engines

  • Blink: Chrome, Edge, Opera, Brave.
  • Webkit: Safari (iOS).
  • Gecko: Firefox.

Most differences are minor today, but some HTML5 features might not be supported in older versions.

2. CSS Reset / Normalize.css

Every browser has its own default styles (User Agent Stylesheet). E.g., Firefox might use 10px margin for body, Chrome uses 8px.

  • Solution: Use a CSS Reset (removes all styles) or Normalize.css (makes them consistent) at the start of your CSS.

3. Polyfills

A polyfill is a piece of JavaScript code that provides modern functionality on older browsers that do not natively support it.

4. Validating Support

Use CanIUse.com to check if a feature (like <dialog> or WebP) is supported by the browsers you target.
