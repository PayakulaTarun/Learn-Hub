HTML Best Practices

Writing code that works is not enough. You must write code that is clean, readable, and standard-compliant.

1. Always Declare Doctype

Ensure <!DOCTYPE html> is the first line to enable standard rendering mode.

2. Use Lowercase Element Names

HTML is case-insensitive, but <body> is cleaner and standard-preferred over <BODY>.

3. Close All Tags

Even if browsers handle unclosed tags (like <li>), always close them to avoid unexpected layout bugs. (XHTML style).

4. Quote Attribute Values

Always use quotes. class="container" vs class=container. Quotes handle spaces correctly.

5. Use Semantic Elements

Don't use <div> if <header>, <footer>, or <button> is more appropriate.

6. Meaningful Alt Text

Always provide alt for images. Leave empty alt="" ONLY if the image is purely decorative.

7. Indentation

Indent your code consistently (2 or 4 spaces) to show nesting clearly.

8. Avoid Inline Styles

Don't use style="color:red". Use external CSS classes. Separates structure (HTML) from design (CSS).
