Python Data Types - Complete Guide

Theory & Concepts

What are Data Types?

Data types define what kind of value a variable can hold and what operations can be performed on it.

Python Built-in Data Types

Python has 14 built-in data types grouped into categories:

1. Numeric Types

Integer (int)

Whole numbers without decimal point.

  • Can be positive, negative, or zero
  • No size limit (limited by memory)
age = 25
temperature = -10
big_number = 999999999999999999

Float (float)

Numbers with decimal point.

  • Can represent fractions
  • Limited precision (~15-17 decimal places)
height = 5.9
pi = 3.14159
temperature = -273.15

Complex (complex)

Complex numbers with real and imaginary parts.

  • Format: a + bj where j = √-1
  • Used in scientific computing
z = 3 + 4j
w = 5j

Comparison Table:

| Type | Example | Description | Use Case | |------|---------|-------------|----------| | int | 100, -5, 0 | Whole numbers | Counts, IDs, ages | | float | 3.14, -0.5 | Decimal numbers | Measurements, prices | | complex | 3+4j | Complex numbers | Math, engineering |

2. Text Type

String (str)

Sequence of characters enclosed in quotes.

  • Single quotes: 'hello'
  • Double quotes: "hello"
  • Triple quotes: '''hello''' or """hello""" (multiline)
name = "Alice"
message = 'Hello, World!'
description = '''This is a
multiline string'''

String Operations:

"Hello" + " World"  # Concatenation: "Hello World"
"Hi" * 3            # Repetition: "HiHiHi"
len("Python")       # Length: 6
"HELLO".lower()     # Convert: "hello"

3. Sequence Types

List (list)

Ordered, mutable collection.

  • Can contain different data types
  • Enclosed in square brackets []
  • Can be modified after creation
numbers = [1, 2, 3, 4, 5]
mixed = [1, "hello", 3.14, True]
empty = []

Tuple (tuple)

Ordered, immutable collection.

  • Cannot be modified after creation
  • Enclosed in parentheses ()
  • Faster than lists
coordinates = (10, 20)
colors = ("red", "green", "blue")
single = (5,)  # Comma needed for single element

Set (set)

Unordered, unique collection.

  • No duplicate values
  • Enclosed in curly braces {}
  • Mutable but elements must be immutable
unique_numbers = {1, 2, 3, 3, 3}  # Stores {1, 2, 3}
fruits = {"apple", "banana", "orange"}

Comparison Table:

| Type | Ordered | Mutable | Duplicates | Syntax | |------|---------|---------|------------|--------| | list | ✅ Yes | ✅ Yes | ✅ Allowed | [1, 2, 3] | | tuple | ✅ Yes | ❌ No | ✅ Allowed | (1, 2, 3) | | set | ❌ No | ✅ Yes | ❌ Not allowed | {1, 2, 3} |

4. Mapping Type

Dictionary (dict)

Key-value pairs, unordered (Python 3.7+ maintains insertion order).

  • Keys must be unique and immutable
  • Values can be any type
  • Enclosed in curly braces with colon {key: value}
person = {
    "name": "Alice",
    "age": 25,
    "city": "New York"
}

scores = {"math": 90, "science": 85}

5. Boolean Type

Boolean (bool)

True or False values.

  • Used in conditions and comparisons
  • Only two values: True and False (capital T and F)
is_active = True
has_license = False

# From comparisons
result = (5 > 3)  # True
equal = (10 == 5)  # False

Truthy and Falsy Values:

Falsy (evaluate to False):

  • False
  • None
  • 0, 0.0
  • Empty collections: '', [], (), {}, set()

Truthy (evaluate to True):

  • Everything else!
  • Non-zero numbers
  • Non-empty collections

6. None Type

NoneType represents absence of value.

  • Only one value: None
  • Similar to null in other languages
result = None
optional_value = None

Mutable vs Immutable

Immutable Types

Cannot be changed after creation. Modifying creates a new object.

Immutable types:

  • int, float, complex
  • str
  • tuple
  • bool
  • frozenset
x = "hello"
x = x + " world"  # Creates NEW string, doesn't modify original

Mutable Types

Can be changed after creation. Modifications happen in-place.

Mutable types:

  • list
  • dict
  • set
my_list = [1, 2, 3]
my_list.append(4)  # Modifies existing list
print(my_list)  # [1, 2, 3, 4]

Why It Matters:

# Immutable (creates new object)
x = 5
y = x
x = 10
print(y)  # 5 (unchanged)

# Mutable (same object)
list1 = [1, 2, 3]
list2 = list1
list1.append(4)
print(list2)  # [1, 2, 3, 4] (changed!)

Type Checking with type()

Use type() function to check data type:

type(100)         # <class 'int'>
type(3.14)        # <class 'float'>
type("hello")     # <class 'str'>
type([1, 2, 3])   # <class 'list'>
type({"a": 1})    # <class 'dict'>

Check if variable is specific type:

x = 100
if type(x) == int:
    print("x is an integer")

# Better way (recommended):
if isinstance(x, int):
    print("x is an integer")

Type Conversion (Type Casting)

Explicit Conversion

Convert between types using constructor functions:

int("123")      # String to int: 123
float("3.14")   # String to float: 3.14
str(100)        # Int to string: "100"
bool(1)         # Int to bool: True
list("abc")     # String to list: ['a', 'b', 'c']
tuple([1,2,3])  # List to tuple: (1, 2, 3)
set([1,2,2,3])  # List to set: {1, 2, 3}

Conversion Table:

| From → To | Function | Example | Result | |-----------|----------|---------|--------| | str → int | int() | int("123") | 123 | | str → float | float() | float("3.14") | 3.14 | | int → str | str() | str(100) | "100" | | float → int | int() | int(3.7) | 3 (truncates) | | list → tuple | tuple() | tuple([1,2]) | (1, 2) | | list → set | set() | set([1,1,2]) | {1, 2} |

Implicit Conversion

Python automatically converts types in some operations:

x = 5       # int
y = 2.5     # float
z = x + y   # z becomes float: 7.5

result = True + 5  # True converts to 1: result = 6

Common Type Operations

Getting Size/Length

len("Python")      # 6
len([1, 2, 3])     # 3
len({"a": 1})      # 1

Checking Membership

5 in [1, 2, 3, 4, 5]         # True
"hello" in "hello world"     # True
"x" not in {"a", "b", "c"}  # True

Getting Min/Max

max([1, 5, 3])     # 5
min([1, 5, 3])     # 1
max("python")      # 'y' (by ASCII value)
