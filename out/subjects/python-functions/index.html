Python Functions - Complete Guide

Theory & Concepts

What is a Function?

A function is a reusable block of code that performs a specific task.

Real-Life Analogy

Think of a coffee machine:

  • Input: Coffee beans, water, sugar
  • Process: Brew coffee
  • Output: Cup of coffee

Same principle:

  • Input: Parameters
  • Process: Function code
  • Output: Return value

Why Use Functions?

Without functions (repetitive):

# Calculate area multiple times
area1 = 5 * 10
area2 = 7 * 3
area3 = 12 * 8

With function (reusable):

def calculate_area(length, width):
    return length * width

area1 = calculate_area(5, 10)
area2 = calculate_area(7, 3)
area3 = calculate_area(12, 8)

Benefits

  1. ✅ Code Reusability: Write once, use many times
  2. ✅ Organization: Break complex code into smaller pieces
  3. ✅ Maintainability: Fix bugs in one place
  4. ✅ Readability: Descriptive function names make code self-documenting
  5. ✅ Testing: Test individual pieces separately

Defining Functions

Syntax

def function_name(parameters):
    """Docstring: what function does"""
    # Function body (indented)
    # Code to execute
    return value  # Optional

Anatomy of a Function

def greet(name):           # 1. def keyword
    """Greets person"""     # 2. Docstring (optional)
    message = f"Hello, {name}!"  # 3. Function body
    return message         # 4. Return statement

Components

  1. def keyword: Defines function
  2. Function name: Follows variable naming rules
  3. Parameters: (name) - inputs (optional)
  4. Colon (:): Marks start of body
  5. Docstring: """...""" - describes function (optional but recommended)
  6. Body: Indented code block
  7. return: Sends value back (optional)

Calling a Function

# Define
def greet(name):
    return f"Hello, {name}!"

# Call
message = greet("Alice")
print(message)  # Hello, Alice!

Execution Flow:

1. Define function (doesn't execute)
2. Call function: greet("Alice")
3. Jump to function code
4. Execute with name="Alice"
5. Return "Hello, Alice!"
6. Assign to message
7. Continue after call

Parameters vs Arguments

Terminology

Parameters: Variables in function definition

def add(a, b):  # a, b are PARAMETERS
    return a + b

Arguments: Values passed when calling

result = add(5, 3)  # 5, 3 are ARGUMENTS

Think: Parameters are placeholders, Arguments are actual values.

Types of Arguments

1. Positional Arguments

Order matters.

def greet(first_name, last_name):
    return f"Hello, {first_name} {last_name}!"

print(greet("John", "Doe"))    # John Doe
print(greet("Doe", "John"))    # Doe John (wrong order!)

2. Keyword Arguments

Use parameter names, order doesn't matter.

def greet(first_name, last_name):
    return f"Hello, {first_name} {last_name}!"

# Order doesn't matter with keywords
print(greet(last_name="Doe", first_name="John"))  # John Doe
print(greet(first_name="Alice", last_name="Smith"))  # Alice Smith

3. Default Arguments

Parameters with default values.

def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

print(greet("Alice"))              # Hello, Alice!
print(greet("Bob", "Hi"))          # Hi, Bob!
print(greet("Charlie", greeting="Hey"))  # Hey, Charlie!

Rules:

  • Default parameters must come after non-default parameters
  • Can override defaults when calling
# ✅ Correct
def func(a, b=10, c=20):
    pass

# ❌ Wrong - non-default after default
def func(a=10, b, c):
    pass

Return Statement

Purpose

Send value(s) back from function.

Syntax

return value

How It Works

def add(a, b):
    result = a + b
    return result  # Send back to caller
    print("After return")  # Never executes!

total = add(5, 3)  # total = 8

Key Points:

  1. return exits function immediately
  2. Code after return doesn't execute
  3. Can return any data type
  4. Can return multiple values (as tuple)

No Return Statement

Functions without return return None.

def greet(name):
    print(f"Hello, {name}!")
    # No return statement

result = greet("Alice")  # Prints: Hello, Alice!
print(result)  # None

Multiple Return Values

Return tuple (automatically unpacked).

def get_student_info():
    name = "Alice"
    age = 20
    grade = 'A'
    return name, age, grade  # Returns tuple

# Unpack
student_name, student_age, student_grade = get_student_info()
print(student_name)  # Alice
print(student_age)   # 20
print(student_grade) # A

Early Return

Use return to exit early.

def is_adult(age):
    if age < 0:
        return None  # Invalid age
    if age >= 18:
        return True
    return False  # age < 18

print(is_adult(25))   # True
print(is_adult(15))   # False
print(is_adult(-5))   # None

Function Scope

Local Scope

Variables inside function exist only there.

def my_function():
    local_var = 10  # Local to function
    print(local_var)  # Works

my_function()
print(local_var)  # Error! local_var doesn't exist here

Global Scope

Variables outside functions.

global_var = 100  # Global

def my_function():
    print(global_var)  # Can read global

my_function()  # 100
print(global_var)  # 100

Modifying Global Variables

count = 0  # Global

def increment():
    global count  # Declare we're using global
    count += 1

increment()
increment()
print(count)  # 2

Best Practice: Avoid global variables, use parameters and return values instead.

Advanced: *args and **kwargs

Variable Number of Arguments

*args: For variable positional arguments

def sum_all(*numbers):
    """Sum any number of arguments"""
    total = 0
    for num in numbers:
        total += num
    return total

print(sum_all(1, 2, 3))          # 6
print(sum_all(10, 20, 30, 40))   # 100
print(sum_all(5))                 # 5

**kwargs: For variable keyword arguments

def print_info(**info):
    """Print key-value pairs"""
    for key, value in info.items():
        print(f"{key}: {value}")

print_info(name="Alice", age=25, city="NYC")
# Output:
# name: Alice
# age: 25
# city: NYC

Combined Example

def student_info(name, *subjects, **details):
    print(f"Name: {name}")
    print(f"Subjects: {subjects}")
    print(f"Details: {details}")

student_info("Alice", "Math", "Science", age=20, grade='A')
# Name: Alice
# Subjects: ('Math', 'Science')
# Details: {'age': 20, 'grade': 'A'}

Lambda Functions (Anonymous)

Short, one-line functions.

Syntax

lambda parameters: expression

Example

# Regular function
def square(x):
    return x ** 2

# Lambda (same thing)
square_lambda = lambda x: x ** 2

print(square(5))         # 25
print(square_lambda(5))  # 25

Common Use: with map(), filter()

numbers = [1, 2, 3, 4, 5]

# Square all numbers
squared = list(map(lambda x: x**2, numbers))
print(squared)  # [1, 4, 9, 16, 25]

# Filter even numbers
even = list(filter(lambda x: x % 2 == 0, numbers))
print(even)  # [2, 4]

Docstrings

Document what function does.

Syntax

def function_name(params):
    """
    Brief description.
    
    Parameters:
        param1 (type): description
        param2 (type): description
    
    Returns:
        type: description
    """
    # code

Example

def calculate_area(length, width):
    """
    Calculate area of rectangle.
    
    Parameters:
        length (float): Length in meters
        width (float): Width in meters
    
    Returns:
        float: Area in square meters
    """
    return length * width

# Access docstring
print(calculate_area.__doc__)

Recursive Functions

Function that calls itself.

Example: Factorial

def factorial(n):
    """Calculate n! = n × (n-1) × ... × 1"""
    if n == 0 or n == 1:
        return 1
    return n * factorial(n - 1)

print(factorial(5))  # 120
# 5 × 4 × 3 × 2 × 1 = 120

Execution trace:

factorial(5)
  = 5 * factorial(4)
  = 5 * (4 * factorial(3))
  = 5 * (4 * (3 * factorial(2)))
  = 5 * (4 * (3 * (2 * factorial(1))))
  = 5 * (4 * (3 * (2 * 1)))
  = 5 * (4 * (3 * 2))
  = 5 * (4 * 6)
  = 5 * 24
  = 120

Important: Always have base case to stop recursion!
