Python Conditional Statements - Complete Guide

Conditional statements are fundamental to programming - they let programs make decisions. **Three Main Types:** ✅ **if** - Execute code only if condition is True ```python if age >= 18: print("Adult") ``` ✅ **if-else** - Two paths (one must execute) ```python if age >= 18: print("Adult") else: print("Minor") ``` ✅ **if-elif-else** - Multiple conditions (first True wins) ```python if score >= 90: grade = 'A' elif score >= 80: grade = 'B' else: grade = 'F' ``` **Key Rules:** 1. **Colon (:)** required after condition 2. **Indentation** (4 spaces) defines code block 3. **Only first True** executes in if-elif-else 4. **Order matters** - specific before general 5. **else optional** - default case **Flow Decision:** | Scenario | Use | |----------|-----| | Single check | `if` | | Two options | `if-else` | | Multiple exclusive options | `if-elif-else` | | Dependent checks | Nested `if` | | Independent checks | Multiple `if` | **Common Patterns:** ```python # Range check if 0 <= score <= 100: # Multiple conditions if age >= 18 and has_license: # Ternary (simple if-else) status = "Adult" if age >= 18 else "Minor" # Nested (dependent) if logged_in: if has_permission: # access granted ``` **Remember:** - **=** assigns, **==** compares - Indent with 4 spaces - Order if-elif from specific to general - Use `and` for all-required, `or` for any-required Master conditionals to build intelligent, responsive programs!

Python Dictionaries - Complete Guide

Dictionaries are Python's key-value data structure - fast, flexible, and essential for structured data. **Key Concepts:** ✅ **Key-Value Pairs**: Data stored as `{key: value}` ✅ **Unique Keys**: No duplicates allowed ✅ **Immutable Keys**: str, int, tuple (not list) ✅ **Any Values**: Lists, dicts, objects, anything ✅ **O(1) Lookup**: Hash table = very fast ✅ **JSON-like**: Perfect for structured data **Creation:** ```python dict = {"name": "Alice", "age": 25} dict = dict(name="Alice", age=25) dict = {k: v for k, v in pairs} ``` **Access:** ```python value = dict["key"] # KeyError if missing value = dict.get("key", default) # Safe ``` **Essential Methods:** | Method | Action | |--------|--------| | `dict[key] = value` | Add/update | | `dict.get(key, default)` | Safe access | | `dict.pop(key)` | Remove & return | | `dict.keys()` | All keys | | `dict.values()` | All values | | `dict.items()` | Key-value pairs | | `dict.update(other)` | Merge dicts | **Iteration (Best Practice):** ```python for key, value in dict.items(): print(f"{key}: {value}") ``` **Nested Dictionaries:** ```python data = { "user1": {"name": "Alice", "age": 25}, "user2": {"name": "Bob", "age": 30} } name = data["user1"]["name"] # Alice ``` **Why Dictionaries are Fast:** - Hash tables: O(1) lookup - List: O(n) search - Dict: instant key-based access **When to Use:** | Scenario | Use | |----------|-----| | Need key-based lookup | ✅ Dict | | Store configuration | ✅ Dict | | JSON-like data | ✅ Dict | | Counting/frequency | ✅ Dict | | Ordered sequence | List | | Just need values | List | **Common Patterns:** ```python # Count frequency freq = {} for item in items: freq[item] = freq.get(item, 0) + 1 # Group by key groups = {} for item in items: key = get_key(item) groups.setdefault(key, []).append(item) # Merge dicts combined = {**dict1, **dict2} ``` **Remember:** ```python # ❌ Wrong dict[[1,2]] = "value" # Lists not hashable! value = dict["missing"] # KeyError! # ✅ Correct dict[(1,2)] = "value" # Tuples OK value = dict.get("missing", default) ``` Master dictionaries for efficient data management!

Python Exception Handling - Complete Guide

Exception handling prevents crashes by catching and managing runtime errors gracefully. **Basic Structure:** ```python try: # Code that might fail risky_operation() except ExceptionType: # Handle specific error handle_error() else: # Only if try succeeds success_code() finally: # Always executes cleanup_code() ``` **Execution Flow:** | Case | try | except | else | finally | |------|-----|--------|------|--------| | Success | ✓ | ✗ | ✓ | ✓ | | Error caught | Partial | ✓ | ✗ | ✓ | | Error not caught | Partial | ✗ | ✗ | ✓ then crash | **Common Exceptions:** ```python ZeroDivisionError # 10 / 0 ValueError # int('abc') TypeError # '5' + 5 IndexError # list[999] KeyError # dict['missing'] FileNotFoundError # open('missing.txt') ``` **Multiple Exceptions:** ```python try: code() except ValueError: # Handle value error except TypeError: # Handle type error except Exception as e: # Catch any other print(f"Error: {e}") ``` **When to Use Each:** - **try**: Wrap risky code - **except**: Handle specific errors - **else**: Code that runs on success - **finally**: Guaranteed cleanup **Raising Exceptions:** ```python def validate(age): if age < 0: raise ValueError("Age cannot be negative") return age # Re-raise try: code() except Exception as e: log(e) raise # Propagate error ``` **Custom Exceptions:** ```python class ValidationError(Exception): pass raise ValidationError("Invalid data") ``` **Best Practices:** ```python # ✅ Specific exceptions try: code() except ValueError: handle_value_error() # ❌ Too broad try: code() except: pass # Don't do this! # ✅ Helpful messages raise ValueError(f"Age must be 0-150, got {age}") # ✅ Order: specific → general try: code() except ValueError: # Specific first pass except Exception: # General last pass ``` **Common Patterns:** ```python # Default value try: value = dict[key] except KeyError: value = default # Retry logic for attempt in range(3): try: return do_operation() except NetworkError: if attempt < 2: time.sleep(1) else: raise # Resource cleanup file = open('data.txt') try: process(file) finally: file.close() ``` **Remember:** - Exceptions for errors, not flow control - Be specific with exception types - Always clean up resources (finally) - Provide helpful error messages - Don't silence errors (log them!) Master exception handling for robust, production-ready code!

Python File Handling - Complete Guide

File handling enables programs to read, write, and persist data - essential for real-world applications. **File Modes:** | Mode | Action | Creates | Overwrites | |------|--------|---------|------------| | `'r'` | Read | ❌ | N/A | | `'w'` | Write | ✅ | ✅ | | `'a'` | Append | ✅ | ❌ | | `'rb'` | Read binary | ❌ | N/A | **The 'with' Statement:** ```python # ✅ Best practice with open('file.txt', 'r') as file: content = file.read() # Auto-closes even if error # ❌ Avoid (manual close) file = open('file.txt', 'r') content = file.read() file.close() # Might not run if error ``` **Reading Methods:** ```python # Entire file content = file.read() # One line line = file.readline() # All lines as list lines = file.readlines() # Best: iterate (memory efficient) for line in file: print(line.strip()) ``` **Writing:** ```python # Write (overwrites) with open('file.txt', 'w') as file: file.write("Hello\n") # Append (preserves) with open('file.txt', 'a') as file: file.write("World\n") ``` **JSON Files:** ```python import json # Write with open('data.json', 'w') as file: json.dump(data, file, indent=4) # Read with open('data.json', 'r') as file: data = json.load(file) ``` **CSV Files:** ```python import csv # Write with open('data.csv', 'w', newline='') as file: writer = csv.writer(file) writer.writerows(rows) # Read with open('data.csv', 'r') as file: reader = csv.reader(file) for row in reader: print(row) ``` **Error Handling:** ```python try: with open('file.txt', 'r') as file: content = file.read() except FileNotFoundError: print("File not found") ``` **Path Operations:** ```python from pathlib import Path # Check existence if Path('file.txt').exists(): # Read file # Read/write text content = Path('file.txt').read_text() Path('file.txt').write_text(content) ``` **Best Practices:** - ✅ Always use `with` statement - ✅ Use 'a' for logs, 'w' for fresh data - ✅ Handle FileNotFoundError - ✅ Use encoding='utf-8' for special chars - ✅ Use pathlib for modern code - ✅ Iterate for large files, not readlines() **Common Patterns:** ```python # Copy file with open('src.txt', 'r') as src: with open('dst.txt', 'w') as dst: dst.write(src.read()) # Append to log with open('log.txt', 'a') as file: file.write(f"{timestamp}: {message}\n") # Process large file with open('big.txt', 'r') as file: for line in file: process(line) ``` Master file handling for data persistence!

Python Object-Oriented Programming (OOP) - Complete Guide

Object-Oriented Programming organizes code around objects - combining data and behavior in reusable, maintainable structures. **Core Concepts:** **Class & Object:** ```python class Car: # Blueprint def __init__(self, brand): self.brand = brand # Instance variable def start(self): # Instance method return f"{self.brand} started" car = Car("Toyota") # Object (instance) ``` **Four Pillars:** 1. **Encapsulation** ```python class Account: def __init__(self, balance): self.__balance = balance # Private @property def balance(self): # Controlled access return self.__balance ``` 2. **Inheritance** ```python class Animal: def speak(self): pass class Dog(Animal): # Dog IS-A Animal def speak(self): return "Woof!" ``` 3. **Polymorphism** ```python # Same method, different implementations for animal in [Dog(), Cat()]: print(animal.speak()) # Different outputs ``` 4. **Abstraction** ```python from abc import ABC, abstractmethod class Shape(ABC): @abstractmethod def area(self): # Must implement pass ``` **Key Features:** | Feature | Purpose | |---------|----------| | `__init__` | Constructor | | `self` | Current instance | | `super()` | Call parent method | | `@property` | Getter | | `@setter` | Setter with validation | | `@classmethod` | Operate on class | | `@staticmethod` | Utility function | **Inheritance:** ```python class Parent: def __init__(self, x): self.x = x class Child(Parent): def __init__(self, x, y): super().__init__(x) # Call parent self.y = y ``` **Encapsulation Levels:** ```python class MyClass: def __init__(self): self.public = 1 # Public self._protected = 2 # Protected (convention) self.__private = 3 # Private (name mangling) ``` **Magic Methods:** ```python class Point: def __str__(self): # Inheritance: IS-A class Dog(Animal): # Dog IS-A Animal pass # Composition: HAS-A class Car: def __init__(self): self.engine = Engine() # Car HAS-A Engine ``` **When to Use OOP:** ✅ Complex systems with related entities ✅ Code reuse through inheritance ✅ Data protection with encapsulation ✅ Modeling real-world objects ✅ Large codebases needing organization **Best Practices:** - Keep classes focused (single responsibility) - Favor composition over deep inheritance - Use properties for controlled access - Document with docstrings - Follow naming conventions - Make attributes private when needed **Common Patterns:** ```python # Singleton class Singleton: _instance = None def __new__(cls): if not cls._instance: cls._instance = super().__new__(cls) return cls._instance # Factory class ShapeFactory: @staticmethod def create(shape_type): if shape_type == 'circle': return Circle() elif shape_type == 'rectangle': return Rectangle() ``` Master OOP for scalable, maintainable code!

Python Operators - Complete Guide

Operators are fundamental to Python programming. They perform operations on data and control program flow. **Types of Operators:** ✅ **Arithmetic**: `+ - * / // % **` - `/` always float, `//` integer division - `%` remainder, `**` power ✅ **Comparison**: `== != > < >= <=` - Return boolean (True/False) - Can chain: `1 < x < 10` ✅ **Logical**: `and or not` - `and`: both true - `or`: at least one true - `not`: inverts ✅ **Assignment**: `= += -= *= /=` - `x += 5` ≡ `x = x + 5` ✅ **Membership**: `in not in` - Check if value in sequence ✅ **Identity**: `is is not` - Check same object - Use `is None`, not `== None` **Critical Distinctions:** | Concept | Correct | Wrong | |---------|---------|-------| | Assignment | `x = 5` | `x == 5` | | Comparison | `x == 5` | `x = 5` | | Value check | `a == b` | `a is b` | | None check | `x is None` | `x == None` | | Integer div | `7 // 2` (3) | `7 / 2` (3.5) | | Remainder | `7 % 3` (1) | | **Operator Precedence (memorize):** ``` ** > */% > +- > comparisons > and > or ``` **Best Practices:** - Use `==` for values, `is` for identity - Use `//` for integer division - Add parentheses for clarity - Check against `None` with `is` **Common Patterns:** ```python # Even/odd check if x % 2 == 0: print("Even") # Range check if 0 <= score <= 100: print("Valid") # Multiple conditions if age >= 18 and has_id: print("Allowed") # Increment count += 1 # Instead of count = count + 1 ``` Master operators to write efficient, readable Python code!

Python Sets - Complete Guide

Sets are Python's implementation of mathematical set theory - unique, unordered collections with powerful operations. **Key Characteristics:** ✅ **Unordered**: No indexing, no guaranteed order ✅ **Unique**: Automatically removes duplicates ✅ **Fast**: O(1) membership testing ✅ **Mutable**: Can add/remove items ✅ **Hashable elements**: Items must be immutable **Creation:** ```python my_set = {1, 2, 3} # With braces my_set = set([1,2,3]) # From list empty = set() # Empty (NOT {}) ``` **Set Operations (Mathematical):** ```python A = {1, 2, 3, 4} B = {3, 4, 5, 6} A | B # Union: {1,2,3,4,5,6} A & B # Intersection: {3,4} A - B # Difference: {1,2} A ^ B # Sym Diff: {1,2,5,6} ``` **Common Methods:** | Method | Action | |--------|--------| | `add(x)` | Add element | | `update(iter)` | Add multiple | | `remove(x)` | Remove (error if missing) | | `discard(x)` | Remove (no error) | | `pop()` | Remove arbitrary | | `clear()` | Remove all | **When to Use Sets:** ✅ Remove duplicates ✅ Membership testing (fast!) ✅ Mathematical operations (union, intersection) ✅ Find common/unique elements ✅ No need for order ✅ Ensure uniqueness **vs Other Types:** | Need | Use | |------|-----| | Order matters | List | | Duplicates needed | List | | Unique items only | **Set** | | Fast membership | **Set** | | Immutable | Tuple/frozenset | | Key-value pairs | Dict | **Common Patterns:** ```python # Remove duplicates unique = list(set(my_list)) # Find common common = set(list1) & set(list2) # Fast lookup allowed = {"red", "green", "blue"} if color in allowed: # O(1) fast! # Count unique unique_count = len(set(items)) ``` **Important:** ```python # ❌ Wrong empty = {} # Dict, not set! my_set[0] # No indexing! my_set = {[1,2]} # Lists not hashable! # ✅ Correct empty = set() list(my_set)[0] # Convert first my_set = {(1,2)} # Tuples OK ``` Master sets for efficient collection operations!

Python Strings - Complete Guide

Strings are one of Python's most used data types - immutable sequences of characters with rich functionality. **Key Concepts:** ✅ **Immutable**: Cannot change after creation ✅ **Sequence**: Ordered, indexed, iterable ✅ **Versatile**: 40+ built-in methods ✅ **Unicode**: Supports all languages **Creation:** ```python single = 'Hello' double = "World" multiline = '''Multiple lines''' raw = r'C:\Users' # Ignores escapes ``` **Indexing & Slicing:** ```python text = "Python" text[0] # 'P' (first) text[-1] # 'n' (last) text[0:3] # 'Pyt' text[::-1] # 'nohtyP' (reverse) ``` **Essential Methods:** | Method | Purpose | |--------|----------| | `upper(), lower()` | Case conversion | | `strip()` | Remove whitespace | | `split()` | String → List | | `join()` | List → String | | `replace(old, new)` | Replace substring | | `find(sub)` | Find index | | `startswith(), endswith()` | Check boundaries | **String Formatting (Modern):** ```python # f-strings (Python 3.6+) name = "Alice" age = 25 f"Name: {name}, Age: {age}" f"Next year: {age + 1}" f"Price: ${19.99:.2f}" # $19.99 f"{1000000:,}" # 1,000,000 ``` **Immutability:** ```python text = "Hello" # ❌ Cannot modify # text[0] = 'J' # TypeError! # ✅ Create new text = 'J' + text[1:] # "Jello" text = text.replace('H', 'J') # "Jello" ``` **Common Patterns:** ```python # Reverse reversed = text[::-1] # Palindrome check is_palindrome = text == text[::-1] # Remove vowels no_vowels = ''.join(c for c in text if c not in 'aeiou') # Word count word_count = len(text.split()) # Join list words = ['Python', 'is', 'awesome'] sentence = ' '.join(words) ``` **Performance Tips:** ```python # ❌ Slow (creates new string each iteration) result = "" for word in words: result += word # ✅ Fast (single join operation) result = ''.join(words) ``` **String Operations:** ```python 'Hello' + ' ' + 'World' # Concatenate 'Ha' * 3 # 'HaHaHa' 'Python' in text # Membership len(text) # Length ``` **Remember:** - Strings are immutable - Use f-strings for formatting - join() not + for building strings - Methods return NEW strings - Slicing [::-1] reverses Master strings for text processing!

Python Tuples - Complete Guide

Tuples are Python's immutable, ordered collections - perfect for data that shouldn't change. **Key Characteristics:** ✅ **Ordered**: Items maintain position ✅ **Immutable**: Cannot be changed after creation ✅ **Hashable**: Can be dict keys ✅ **Fast**: Faster than lists ✅ **Memory efficient**: Use less memory **Creation:** ```python my_tuple = (1, 2, 3) # With parentheses my_tuple = 1, 2, 3 # Without (packing) single = (5,) # Trailing comma! empty = () ``` **Tuple vs List:** | Feature | Tuple | List | |---------|-------|------| | Syntax | `()` | `[]` | | Mutable | ❌ No | ✅ Yes | | Speed | ✅ Fast | Slower | | Memory | ✅ Less | More | | Methods | 2 | 11+ | | Dict key | ✅ Yes | ❌ No | **Methods (Only 2!):** ```python my_tuple.count(value) # Count occurrences my_tuple.index(value) # Find first index ``` **Packing & Unpacking:** ```python # Packing coords = 10, 20 # Creates (10, 20) # Unpacking x, y = coords # x=10, y=20 # Extended first, *rest, last = (1,2,3,4,5) # first=1, rest=[2,3,4], last=5 # Swapping a, b = b, a ``` **When to Use:** **Use Tuples:** - GPS coordinates: `(lat, lon)` - RGB colors: `(255, 0, 0)` - Database rows - Function returns - Fixed configuration - Dictionary keys **Use Lists:** - Shopping cart (add/remove) - To-do list (modify) - User input - Sorted results **Important:** ```python # ❌ Cannot modify my_tuple[0] = 10 # TypeError! # ✅ Can create new new = my_tuple + (10,) # ❌ Single element single = (5) # int! # ✅ Correct single = (5,) # tuple ``` **Common Patterns:** ```python # Multiple returns def func(): return x, y, z a, b, c = func() # Coordinate pairs locations = [(x1,y1), (x2,y2)] # Swap a, b = b, a ``` **Remember:** - Tuples = immutable, faster, hashable - Lists = mutable, flexible - Use tuples for data that won't change - Trailing comma for single element! Choose tuples for data integrity and performance!

Python Virtual Environments - Complete Guide

Virtual environments are essential for professional Python development - they isolate project dependencies and prevent conflicts. **What is Virtual Environment:** - Isolated Python installation with own packages - Separate from system Python - One venv per project - Prevents dependency conflicts **Why Essential:** ``` Without venv: Project A: Django 2.2 Project B: Django 4.0 Global Python: Only ONE version! ❌ With venv: Project A venv: Django 2.2 ✅ Project B venv: Django 4.0 ✅ Both work! No conflicts! ✅ ``` **Create Virtual Environment:** ```bash python -m venv venv ``` **Activate:** ```bash # Windows CMD venv\Scripts\activate.bat # Windows PowerShell venv\Scripts\Activate.ps1 # Mac/Linux source venv/bin/activate # Activated: (venv) shows in prompt ``` **Deactivate:** ```bash deactivate ``` **Complete Workflow:** ```bash # 1. Create project mkdir my_project && cd my_project # 2. Create venv python -m venv venv # 3. Activate source venv/bin/activate # or Windows equivalent # 4. Install packages (venv) $ pip install requests pandas # 5. Save dependencies (venv) $ pip freeze > requirements.txt # 6. Work on project (venv) $ python main.py # 7. Deactivate (venv) $ deactivate ``` **requirements.txt:** ```bash # Create pip freeze > requirements.txt # Install pip install -r requirements.txt # Ensures same packages across team/deployments ``` **Check if Active:** 1. Prompt shows `(venv)` 2. `which python` → points to venv 3. Python: `sys.prefix != sys.base_prefix` **Best Practices:** - ✅ One venv per project - ✅ Add venv/ to .gitignore - ✅ Commit requirements.txt - ✅ Activate before pip install - ✅ Use descriptive venv names - ❌ Never commit venv folder - ❌ Never install in global Python **.gitignore:** ``` venv/ env/ *.pyc __pycache__/ ``` **OS-Specific Commands:** | OS | Create | Activate | |----|--------|----------| | Windows CMD | `python -m venv venv` | `venv\Scripts\activate.bat` | | Windows PS | `python -m venv venv` | `venv\Scripts\Activate.ps1` | | Mac/Linux | `python -m venv venv` | `source venv/bin/activate` | **Common Issues:** **PowerShell:** ```powershell # Error: running scripts disabled Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser ``` **Forgot to activate:** ```bash # ❌ No (venv) in prompt $ pip install package # Installs globally! # ✅ Activate first $ source venv/bin/activate (venv) $ pip install package # Safe ``` **Benefits:** 1. **Isolation**: Each project independent 2. **No conflicts**: Different package versions 3. **Clean global**: System Python uncluttered 4. **Reproducible**: requirements.txt recreates environment 5. **Testing**: Test different Python/package versions 6. **Deployment**: Exact dependencies in production **Remember:** - Always use virtual environments - Never commit venv/ folder - Always create requirements.txt - One venv = one project - Activate before installing Master virtual environments for professional Python development!
