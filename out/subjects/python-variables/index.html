Python Variables - Complete Beginner Guide

Theory & Concepts

What is a Variable?

A variable is a named container that stores a value in computer memory. Think of it as a labeled box where you can put information and retrieve it later.

Real-Life Analogy

Imagine variables as labeled storage containers:

  • Container name (variable name): "cookie_jar"
  • Contents (value): 25 cookies
  • You can:
    • Put things in (assign value)
    • Check what's inside (read value)
    • Replace contents (reassign value)
    • Add/remove items (modify value)

Why Use Variables?

1. Store Data

age = 25  # Store age for later use

2. Reuse Values

price = 100
total = price + price + price  # Reuse price

3. Make Code Readable

# Bad:
print(3.14159 * 5 * 5)

# Good:
pi = 3.14159
radius = 5
area = pi * radius * radius
print(area)

4. Update Values Easily

user_name = "Alice"
user_name = "Bob"  # Easy to change

Creating Variables in Python

Basic Syntax

variable_name = value

No type declaration needed! Python automatically determines the type.

Examples:

name = "Alice"      # String (text)
age = 25            # Integer (whole number)
height = 5.6        # Float (decimal)
is_student = True   # Boolean (True/False)

Dynamic Typing

Python is dynamically typed - you don't declare types, and variables can change types.

What This Means:

Other Languages (Static Typing - Java, C++):

int age = 25;        // Must declare type 'int'
age = "twenty";      // ERROR! Can't change type

Python (Dynamic Typing):

age = 25            # age is integer
age = "twenty"      # Now age is string - ALLOWED!

Type Checking

Use type() to check a variable's type:

x = 100
print(type(x))  # <class 'int'>

x = "hello"
print(type(x))  # <class 'str'>

Variable Naming Rules

MUST Follow (Syntax Rules)

✅ Allowed:

  • Letters: a-z, A-Z
  • Numbers: 0-9 (NOT at start)
  • Underscore: _

❌ NOT Allowed:

  • Cannot start with number: 2name ✗
  • Cannot use spaces: user name ✗
  • Cannot use special characters: user@name, price$ ✗
  • Cannot use keywords: if, for, while, class ✗

Python Keywords (Reserved Words)

These cannot be used as variable names:

False, True, None, and, or, not, if, else, elif,
for, while, break, continue, def, class, return,
import, from, as, try, except, finally, with, etc.

Check if a word is a keyword:

import keyword
print(keyword.iskeyword('for'))  # True
print(keyword.iskeyword('name'))  # False

Naming Conventions (Style Guidelines - PEP 8)

1. Snake Case (Recommended for Variables)

user_name = "Alice"       # Good
first_name = "Bob"        # Good
total_price = 100         # Good

userName = "Alice"        # Not Pythonic (camelCase)
UserName = "Alice"        # Not for variables (PascalCase)

2. Descriptive Names

# Bad - unclear
x = 25
a = "New York"

# Good - descriptive
age = 25
city = "New York"

3. Constants in UPPERCASE

PI = 3.14159
MAX_SIZE = 100
API_KEY = "abc123"

4. Start with Lowercase

name = "Alice"      # Good (variable)
Name = "Alice"      # Looks like class name

5. Meaningful Names

# Bad
a = 5
b = 10
c = a * b

# Good
length = 5
width = 10
area = length * width

Multiple Assignment

Assign Same Value to Multiple Variables

x = y = z = 0
print(x, y, z)  # 0 0 0

Assign Different Values at Once

a, b, c = 1, 2, 3
print(a)  # 1
print(b)  # 2
print(c)  # 3

Practical Use - Swapping

x = 5
y = 10

# Swap values in one line!
x, y = y, x

print(x)  # 10
print(y)  # 5

Variable Reassignment

Variables can be reassigned new values anytime:

score = 0
print(score)  # 0

score = 50
print(score)  # 50

score = score + 25
print(score)  # 75

# Shorthand
score += 10
print(score)  # 85

Variables as References

Variables store references to objects in memory.

For Immutable Types (int, str, float):

x = 5
y = x  # y gets a copy of value
x = 10
print(y)  # 5 (unchanged)

For Mutable Types (list, dict):

list1 = [1, 2, 3]
list2 = list1  # Both point to SAME list
list1.append(4)
print(list2)  # [1, 2, 3, 4] (changed!)

To create independent copy:

list1 = [1, 2, 3]
list2 = list1.copy()  # Creates new list
list1.append(4)
print(list2)  # [1, 2, 3] (unchanged)

Variable Scope (Introduction)

Global Variables - defined outside functions, accessible everywhere Local Variables - defined inside functions, only accessible there

x = 10  # Global variable

def my_function():
    y = 5  # Local variable
    print(x)  # Can access global x
    print(y)  # Can access local y

my_function()
print(x)  # Works
print(y)  # ERROR! y doesn't exist here

(Scope will be covered in detail in Functions chapter)

Special Variables

Underscore Variables

Single Underscore _ - Last result in interactive mode

>>> 5 + 3
8
>>> _
8
>>> _ * 2
16

Leading Underscore _name - Internal/private variable (convention)

_internal_value = 42  # Indicates "don't use outside this module"

Double Underscore __name - Name mangling (avoid in beginner code)

Double Underscore Both Sides __name__ - Magic/dunder methods

__name__  # Special variable (module name)
